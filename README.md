# 表格文件处理工具

一个基于 `tkinter + pandas + openpyxl + tkinterdnd2` 的桌面程序，用于整理多个表格文件。

## 功能

- 左侧数据源分为两块：`老数据` 和 `新数据`
- 支持把文件直接拖到对应的数据源区域导入
- 支持按 sheet 成对操作：双击老数据和新数据中的 sheet，可指定本次只处理这一对 sheet
- 预览区域加大，便于查看字段和值
- 支持识别“无表头新数据”，按老数据模板自适应匹配字段
- 支持识别手机号/电话这类新增字段，只要老数据模板里已有该列
- 导入新数据后可手工确认字段匹配，自动建议不准时可以直接改
- 如果新旧数据表头完全一致，程序会自动确认匹配并跳过确认窗口
- 支持主键匹配合并
  - `更新并新增`
  - `仅更新不新增`
  - `仅用新数据补全老数据空值`
- 支持筛选、去重、批量更新、字段显示/隐藏、导出
- 支持 `xlsx`、`xlsm`、`csv`、`tsv`

## 典型用法

如果你有两份表：

- 老数据：账号、姓名、旧手机号
- 新数据：账号、最新手机号、最新地址

操作方式：

1. 把旧表导入到 `老数据`
2. 把新表导入到 `新数据`
3. 在“主键合并”里选择 `账号`
4. 策略选 `更新并新增`
5. 点击 `应用处理`

结果：

- 老数据里已存在的账号，会按字段比较，只更新有变化的字段
- 新数据里存在但老数据没有的账号，在 `更新并新增` 模式下会追加成新行
- 如果策略改成 `仅更新不新增`，则只更新已存在账号，不新增不存在的账号
- 如果新数据没有标准表头，程序会优先按老数据字段类型自动识别并映射，只保留老数据模板里已有的字段
- 如果老数据后续新增了 `手机号/电话` 字段，而新数据没有表头，只要那一列值看起来像手机号，程序也会自动匹配进去
- 在“字段匹配”窗口里，你可以把新数据某一列手工指定为 `邮箱 / 手机号 / 密码`，也可以直接忽略不需要导入的列

如果你只想补空值、不替换已有值，则把策略改成 `仅用新数据补全老数据空值`。

## 下载即用

- 如果只是使用软件，优先从 GitHub Releases 下载 `wenjian-chuli-windows-x64-v*.zip`
- 解压后双击 `启动表格处理工具.bat` 或 `启动表格处理工具.vbs`
- Release 便携包已包含 Python 运行时、`pandas`、`openpyxl`、`tkinterdnd2` 和拖拽所需资源，不需要额外安装 Python

## 安装源码环境

```powershell
python -m pip install -r requirements.txt
```

## 启动

双击以下任一文件：

- `启动表格处理工具.vbs`
- `启动表格处理工具.bat`

或者手动运行：

```powershell
python main.py
```

## 构建便携版发布包

安装打包依赖：

```powershell
python -m pip install -r requirements-build.txt
```

执行完整构建：

```powershell
python scripts/build_release.py
```

构建脚本会自动执行：

- 单元测试
- 编译检查
- PyInstaller 打包
- 生成 `release/wenjian-chuli-windows-x64-v*.zip`

## 自动发布

- 推送到 `main` 分支会在 GitHub Actions 上自动构建 Windows 便携包
- 推送 `v*` 标签会在构建成功后自动发布 GitHub Release 并上传 zip 资产
